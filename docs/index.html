Iterable

Wraps any JavaScript iterable (i.e. an object that has a Symbol.iterator property) to provide a common iterface, regardless of the underlying concrete type. Where possible and apporpriate this interface matches the built-in Array. Thus, you can call the same Iterable#map on an Array, a MarkLogic Seqeunce, or a generator—anything that’s iterable. Type-specific Iterable extensions override the built-in methods to provide implementations that delegate to their wrapped concrete types.

Iterable(iterable: ESIterable): Iterable
Parameters
iterable (ESIterable) Any iterable (i.e. something that has a Symbol.iterator property)
Returns
Iterable: An new Iterable instance the wraps the passed in iterable
Example
Iterable(
  cts.collections() // Any iterable, such as Array, Map, generator function, etc.
)
  .filter(c => true) // Not very discriminating
  .map(c => { return { name: c, count: cts.frequency(c) }})
  .sort((a, b) => b.count - a.count)
  .reduce((prev, item) => prev + '\r' + item.count + ': ' + item.name, '');

▾ isIterable(obj, ignoreStrings?)

Whether an object is iterable. Only checks for Symbol.iterator. This won’t catch the case where a function implicitly returns a duck-typed iterator. Note that strings are iterable by defintion in the language spec.

isIterable(obj: any, ignoreStrings: boolean?): boolean
Parameters
obj (any) Any object, including null or undefined
ignoreStrings (boolean?) Don’t consider a string iterable. Be careful how you employ this.
Returns
boolean: Whether the object is iterable
Example
Iterable.isIterable([1, 2, 3]);        // true
Iterable.isIterable('asdf');           // true, unfortunately
Iterable.isIterable('asdf', true)      // false
Iterable.isIterable((function*(){})()) // true
Iterable.isIterable({a: 'A'})          // false

▾ reduce(reducer, init)

Accumulate an aggregate value over all of an Iterable instance’s items.

reduce(reducer: function, init: any): any
Parameters
reducer (function) The function that takes the previous value and the current item and returns a new value for the next iteration.
  • prev (any) The accumulated value returned from the previous iteration
  • item (any) The current item
  • index (number) The current index
  • self (Iterable)
init (any) The initial value
Returns
any: The accumlated value
Example
Iterable([1, 2, 3])
  .reduce((prev, item) => prev + item, 0);

// Second iteration: (1, 2) => 3
// Third iteration:  (3, 3) => 6

▾ slice(begin?, end?)

Returns a shallow copy of a portion of the Iterable into a new IterableSequence object selected from begin to end, not including end. slice does not modify the original IterableSequence.

The implementation delegates to fn.subsequence() under the covers.

slice(begin: number?, end: number?): IterableSequence
Parameters
begin (number?) Zero-based start index. Only positive integers are supported.
end (number?) Zero-based end index, not inclusive.
Returns
IterableSequence: A shallow copy of the sliced Sequence
Example
const seq = xdmp.databases();      // Sequence<xs.unsignedLong>
const itr = IterableSequence(seq); // IterableSequence<Sequence<xs.unsignedLong>>
itr
 .slice(2, 4)                      // IterableSequence<xs.unsignedLong>

ESIterable

Iterable is a protocol which when implemented allows a JavaScript object to define their iteration behavior, such as what values are looped over in a for..of loop or iterall's forEach function. Many built-in types implement the Iterable protocol, including Array and Map.

While described by the ES2015 version of JavaScript it can be utilized by any version of JavaScript.
